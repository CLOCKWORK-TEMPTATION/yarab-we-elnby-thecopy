# توثيق تطبيق لوحة معلومات مقاييس النظام

## نظرة عامة على التطبيق

تُعدّ صفحة لوحة معلومات مقاييس النظام (System Metrics Dashboard) صفحة ويب متخصصة في عرض ومراقبة مقاييس النظام بشكل شامل وفوري. تم تصميم هذا التطبيق ليكون واجهة مركزية permettent للمستخدمين متابعة أداء النظام ومتطلباته من حيث استخدام المعالج والذاكرة ومساحة التخزين ومقاييس الشبكة، مما يُمكّنهم من اتخاذ قرارات مستنيرة بشأن إدارة الموارد وتحسين الأداء.

يهدف هذا التطبيق بشكل أساسي إلى توفير رؤية شاملة وفورية لحالة النظام، حيث يعرض البيانات بشكل جذاب بصريًا وسهل الفهم، مما يُساعد مسؤولي النظام والمطورين على مراقبة الصحة العامة للبنية التحتية التقنية. يدعم التطبيق التحديث التلقائي للبيانات لضمان وصول المستخدمين إلى أحدث المعلومات دون الحاجة إلى إعادة تحميل الصفحة يدويًا.

---

## الهيكل التقني للتطبيق

### البنية الأساسية

يقع هذا التطبيق ضمن إطار عمل Next.js الذي يُعدّ من أحدث وأشهر أُطر العمل لتطوير تطبيقات الويب باستخدام React. يعتمد التطبيق على نمط التوجيه الديناميكي (Dynamic Rendering) الذي يُتيح معالجة الطلبات في الوقت الفعلي دون الحاجة إلى إنشاء صفحات ثابتة مسبقًا، وهذا أمر بالغ الأهمية للتطبيقات التي تتطلب بيانات حية ومتجددة باستمرار.

يستخدم التطبيق وضع العميل (Client-Side Rendering) من خلال توجيه `"use client"` الذي يُخبر Next.js بأن هذا المكون يجب تنفيذه على جانب العميل وليس على جانب الخادم. هذا الخيار ضروري نظرًا لاعتماد المكون على مكتبات مثل React Query التي تتطلب وجود كائن QueryClient متاحًا على مستوى المتصفح.

### تكوين التصدير والتصميم

يتميز التطبيق بإعدادات تصدير خاصة تُحدد سلوك العرض والسرعة، حيث تم تعيين `export const dynamic = 'force-dynamic'` لمنع المعالجة المسبقة (Prerendering) أثناء عملية البناء، وهذا ضروري لأن لوحة المعلومات تستخدم React Query hooks التي تتطلب QueryClient متاحًا. كما تم تعيين `export const runtime = 'nodejs'` لتحديد بيئة تشغيل Node.js للتطبيق.

```tsx
export const dynamic = 'force-dynamic';
export const runtime = 'nodejs';
```

## هيكل المكونات

### المكون الرئيسي

المكون الرئيسي للتطبيق هو `SystemMetricsDashboard` الذي يتم استيراده من المسار `@/components/ui/system-metrics-dashboard`. هذا المكون يُعدّ حجر الزاوية في التطبيق وهو المسؤول عن:

- استرجاع البيانات من مصادرها المختلفة
- معالجة وتنسيق البيانات للعرض
- إدارة حالة التحميل والأخطاء
- عرض واجهة المستخدم الرسومية

### هيكل الاستيراد

يستخدم التطبيق استيرادًا نسبيًا محددًا من المجلد `@/components/ui/`، وهذا يُشير إلى وجود مجلد مكونات موحد (Shared Components) داخل المشروع. يُعدّ هذا النمط من التنظيم من أفضل الممارسات في تطوير تطبيقات Next.js，因为它 يُسهّل إعادة استخدام المكونات عبر صفحات متعددة ويضمن تناسق التصميم عبر التطبيق.

## التبعيات والمكتبات

### React Query

يعتمد التطبيق بشكل كبير على مكتبة React Query (المعروفة أيضًا باسم TanStack Query) لإدارة حالة جلب البيانات. توفر هذه المكتبة ميزات أساسية تشمل:

- **التخزين المؤقت التلقائي (Automatic Caching)**: تُخزّن البيانات مؤقتًا لتقليل طلبات الشبكة المتكررة
- **تحديث البيانات في الخلفية (Background Refetching)**: تُحدّث البيانات تلقائيًا دون تدخل المستخدم
- **إدارة حالات التحميل (Loading States)**: توفر حالات واضحة للتحميل والانتقال
- **معالجة الأخطاء (Error Handling)**: تُدير الأخطاء بشكل موحد وتوفر واجهة برمجة تطبيقات بسيطة للتعامل معها

### بيئة التشغيل

يتطلب التطبيق بيئة تشغيل Node.js كما هو محدد في إعدادات التصدير، وهذا يضمن توافقًا مع مكتبات معينة قد لا تكون متاحة في بيئات أخرى مثل Edge Runtime.

## إرشادات الاستخدام

### طريقة الوصول

يمكن الوصول إلى صفحة لوحة المعلومات من خلال المسار `/metrics-dashboard` أو أي مسار فرعي تم تكوينه في نظام التوجيه. بمجرد الوصول إلى الصفحة، سيقوم التطبيق تلقائيًا بجلب البيانات وعرضها للمستخدم.

### التفاعل مع البيانات

يمكن للمستخدمين التفاعل مع لوحة المعلومات من خلال:

- **الاطلاع على المقاييس**: عرض القيم الحالية لجميع مقاييس النظام
- **مراقبة التغييرات**: متابعة التغيرات في الوقت الفعلي
- **تحليل الاتجاهات**: فهم أنماط استخدام الموارد عبر الوقت

---

## اعتبارات الأداء

### التحسينات المُطبّقة

يستخدم التطبيق عدة تقنيات لتحسين الأداء:

- **العرض الديناميكي**: يُجنب إنشاء صفحات ثابتة غير ضرورية
- **الذاكرة المؤقتة**: تُقلل من استهلاك الشبكة والوقت اللازم لجلب البيانات
- **تحديثات جزئية**: تُحدث فقط الأجزاء التي تتغير من الواجهة

### أفضل الممارسات

لضمان الأداء الأمثل، يُنصح بما يلي:

- تجنب إضافة مكونات ثقيلة غير ضرورية داخل لوحة المعلومات
- تحسين استعلامات البيانات لتقليل حجم التحميل
- استخدام مؤشرات تفاعلية لتوضيح حالة البيانات

---

## الصيانة والتطوير المستقبلي

### نقاط الاهتمام

عند تطوير أو صيانة هذا التطبيق، يجب مراعاة:

- **تحديث التبعيات**: التأكد من توافق إصدارات React Query مع باقي المشروع
- **إدارة المفاتيح API**: إذا كان التطبيق يتطلب مفاتيح API، يجب تأمينها بشكل صحيح
- **معالجة الأخطاء**: إضافة معالجة أخطاء شاملة لجميع حالات الفشل المحتملة

###قابلية التوسع

صُمم التطبيق ليكون قابلاً للتوسع، حيث يمكن إضافة مقاييس جديدة بسهولة من خلال:

- إنشاء مكونات جديدة لعرض مقاييس إضافية
- توسيع نطاق API لجلب بيانات أكثر
- إضافة ميزات تصدير وتخصيص للمستخدم

## الخلاصة

تُشكّل صفحة لوحة معلومات مقاييس النظام جزءًا أساسيًا من البنية التحتية للمراقبة، حيث توفر رؤية شاملة وفورية لحالة النظام. يعتمد التطبيق على تقنيات حديثة مثل Next.js وReact Query لتقديم تجربة مستخدم سلسة وفعالة، مع التركيز على الأداء وقابلية الصيانة والتوسع.
