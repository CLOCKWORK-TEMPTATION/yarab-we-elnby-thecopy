# Contract: Confidence Policy

## Purpose

هذا العقد يحدد كيف يتحول نقص الأدلة أو نقص البيئة أو جزئية التغطية إلى
`review mode`
و
`confidence level`
بدل أن يتحول إلى صمت أو ثقة زائفة.

## Review Modes

| Review Mode | Trigger |
|-------------|---------|
| `Static Analysis Only` | لا يوجد أي فحص آلي منفذ بنجاح، أو جميع الفحوصات محجوبة |
| `Partial Execution Review` | بعض الفحوصات نفذت، لكن توجد عناصر محجوبة أو تغطية جزئية أو نقص بيئة مؤثر |
| `Full Execution Review` | جميع الفحوصات الأربعة منفذة، ولا توجد تغطية جزئية أو حجب مؤثر على النطاق الرسمي |

## Confidence Levels

| Confidence | Trigger | Required Rationale |
|------------|---------|--------------------|
| `High` | تشغيل كامل مع تغطية كاملة ودون حجب مؤثر | يذكر أن الحكم مدعوم بتشغيل كامل ونطاق مكتمل |
| `Medium` | تشغيل جزئي أو تغطية جزئية أو حجب محدود التأثير | يذكر الجزء المنفذ وما الذي خفّض الثقة |
| `Low` | غياب بيئة، أو حجب واسع، أو اعتماد شبه كامل على القراءة الساكنة | يذكر بوضوح أن مناطق مهمة لم يمكن تقييمها |

## Mandatory Inputs

| Input | Description |
|-------|-------------|
| `AutomatedCheckResult[]` | نتائج `lint`, `type-check`, `test`, `build` بعد التطبيع |
| `AuditTarget[]` | سجل الأهداف الرسمي بحالات التغطية |
| `environmentBlockers[]` | نواقص البيئة أو المتغيرات أو الملفات الحرجة |
| `manualConstraints[]` | أي قيود تشغيلية تقلل قوة الحكم النهائي |

## Decision Rules

1. لا يجوز أن يرتفع الحكم إلى

```text
High
```

إذا وجد أي

```text
blocked
```

واسع الاتساع أو أي

```text
not_present
```

غير مفسر داخل النطاق الرسمي.
2. تضييق تغطية الواجهة أو الخلفية يخفض الثقة إلى

```text
Medium
```

على الأقل حتى مع نجاح الفحص نفسه.
3. غياب البيئة أو المتغيرات اللازمة للتشغيل يخفض الثقة إلى

```text
Low
```

إذا منع تقييم محور أمني أو تكاملي أو بنائي أساسي.
4. يجب أن يحتوي الخرج النهائي دائمًا على سبب نصي صريح لكل خفض ثقة.
5. لا يجوز استخدام تعبيرات عامة من نوع

```text
good confidence
```

أو

```text
looks healthy
```

من دون سبب قابل للتحقق.
