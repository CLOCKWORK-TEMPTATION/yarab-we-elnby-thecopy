# Agent Context Contract

## Purpose

هذا العقد يحدد كيف تُستهلك الخطة التنفيذية لتحديث سياق الوكيل في:

```text
E:\yarab we elnby\the copy\CLAUDE.md
```

## Inputs from Plan

- `Language/Version`
- `Primary Dependencies`
- `Storage`
- `Testing Framework`
- `Target Platform`
- `Project Type`
- الفرع النشط

## Output Target

يجب أن يؤدي التحديث إلى:

- إضافة أو تحديث معلومات التقنيات النشطة.
- إضافة أو تحديث سجل التغييرات الحديثة.
- الحفاظ على قابلية قراءة الملف الناتج وعدم إدخال placeholders.

## Preservation Rules

- لا يجوز حذف الإرشادات اليدوية الموجودة مسبقًا في الملف الجذري.
- أي قسم جديد يجب أن يضاف تكميليًا لا أن يستبدل العقد اليدوي القائم.
- إذا كان الملف موجودًا أصلًا، فالتحديث يجب أن يكون
  `upsert`
  لا
  `overwrite-from-scratch`

## Validation Rules

- الملف الناتج يجب أن يكون بلا placeholders غير محلولة.
- يجب أن يلتقط التحديث الفرع النشط الحالي.
- إذا غابت الحقول الحرجة من الخطة، يجب أن يفشل التحديث بدل الكتابة الصامتة.

## Transitional Rule

يقرأ المحدث الآن جدول
`Technical Context`
بوصفه المصدر الأول للحقول الحرجة.

وإذا غابت الحقول المطلوبة من الجدول، تعتمد الدورة على قسم:

```text
2.1 Agent Context Seed
```

داخل
`plan.md`
بوصفه fallback انتقالية لا بديلًا دائمًا.

إذا غابت الحقول الحرجة من المصدرين معًا، يجب أن يفشل التحديث صراحة بدل الكتابة
الصامتة.
